1. Создание ссылок:
 
- Чтобы создать файл «file1» использую команду cat > file1;
- Чтобы скопировать «file1» в «file2» использую команду cp file1 file2;
- Чтобы создать символическую ссылку «file3» на «file1» использую команду ln -s file1 file3;
- Чтобы создать жёсткую ссылку «file4» на «file1» использую команду ln file1 file4 ;
- Чтобы посмотреть какие айноды у файлов использую команду ls -li.
	Айноды у «file1» и «file4» одинаковые т.к. «file4» создавался как жёсткая ссылка на «file1».
	У «file2» айнод отличается от «file1».
	У «file3» так же отличается айнод от «file1», потому что содержит указатель на «file1»;
- Чтобы удалить «file1» использую команду rm file1.
	После удаления «file1», с «file2» ничего не изменилось, символическая ссылка «file3» стала ссылаться на несуществующий файл, а также остался один объект dentry 	с указателем на айнод «file4»;

2. Работа со ссылками: 

- Чтобы переименовать созданные файлы использую команду mv file2 renamedfile2 и т.д. ;
- Чтобы создать новую символическую ссылку использую команды ln -s renamedfile2 someLinkRel и ln -s $(pwd)/renamedFile2 someLinkAbs(с указанием абсолютного пути).
- Чтобы переместить ссылки в другую директорию, сначала использовал команду для создания директории mkdir links, а потом чтобы переместить ссылки в эту директорию использовал команду mv some* links/ 
 
	После перемещения символических ссылок в другую директорию осталась рабочей только одна ссылка, которая была создана с указанием 	абсолютного пути.)

3. Права файлов:

- Чтобы создать два произвольных файла использовал команду touch file1 file2;
- Чтобы первому файлу присвоить права на чтение, запись для владельца и группы использовал команду chmod 664 file1;
- Чтобы второму файлу присвоить права на чтение, запись только для владельца использовал команду chmod 644 file2;

4. Создать пользователя, обладающего возможностью выполнят действия от имени суперпользователя.

- Чтобы создать  пользователя использую команду sudo useradd (имя_пользователя);
- Чтобы создать пароль для пользователя использую команду sudo passwd (имя_пользователя);
- Чтобы войти от имени пользователя использую команду su - (имя_пользователя)
